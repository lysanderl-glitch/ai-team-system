# 模板使用指南 Quick Start

> 本指南适用于 Janusd 全体项目团队成员。阅读时间：5 分钟。

## 一、模板库概览

本模板库包含 41+ 个标准 PM 模板，按项目生命周期展开：

| 阶段 | 主要模板 | 谁使用 |
|------|----------|--------|
| 00_售前 | 成本评估表、现场调研模板 | 售前团队 |
| 01_项目启动 | 项目章程（自动生成）、干系人登记册 | PM |
| 02_项目规划 | 策划总表、甘特图、WBS、风险册等 | PM |
| 03_项目执行 | 施工日报、UAT测试用例、验收方案 | 工程师/测试工程师 |
| 04_项目监控 | 周报、状态报告、变更请求表等 | PM |
| 05_项目收尾 | 经验教训总结 | PM + 团队 |
| 财务工具 | EVM仪表板、结算模板 | PM + 财务 |

## 二、模板使用流程

### 新项目启动对照清单
- [ ] 售前：填写 JDG-PRE-LOG-002 售前成本评估表
- [ ] 签约后：销售在项目注册表填写信息，状态设为「已签约」
- [ ] WF-01 自动生成：项目章程 + Asana项目 + Slack通知
- [ ] PM填写 JDG-INI-ORG-002 干系人登记册
- [ ] PM完善 JDG-PLN-PLN-001 项目策划总表

### 每周循环操作
- [ ] 周一：填写 JDG-MON-STS-013 项目周报
- [ ] 周五：检查风险登记册 JDG-MON-RSK-015
- [ ] 如有变更：提交 JDG-MON-CHG-011 变更请求表

### 每月监控
- [ ] 更新 JDG-MON-FIN-016 成本监控台账
- [ ] 发布 JDG-MON-STS-010 项目状态报告

### 项目收尾
- [ ] 完成 JDG-ACP-RXD-001 项目验收策划方案
- [ ] 完成 JDG-CLO-RXD-001 经验教训总结

## 三、模板命名规范
```
JDG - [STAGE] - [TYPE] - [SEQ] _ 名称 _ V版本号
```

阶段代码: PRE/INI/PLN/EXE/MON/CLO/ACP/CAR/FIN
文件类型: CHG/COM/FIN/LOG/MTG/ORG/PLN/PRO/QTY/RES/RSK/RXD/SCP/SPC/STD/STS/TRN/WBS

## 四、自动化工具
| 功能 | 方式 |
|------|------|
| 项目章程自动生成 | 注册表状态改为「已签约」，WF-01 自动处理 |
| 周报自动生成 | 每周一 09:00 WF-04 推送 Slack |
| 逐期预警 | 每日 09:00 WF-05 检查 |
| 会议纪要自动化 | Fireflies 转录 + WF-07 |

## 五、常见问题
- 模板更新 -> 编辑Notion页面，更新版本号，通知PMO
- 下载Excel/Word -> 模板页面底部查看本地文件路径
- CR审批时长 -> C1<=1天; C2<=3天; C3<=5天
