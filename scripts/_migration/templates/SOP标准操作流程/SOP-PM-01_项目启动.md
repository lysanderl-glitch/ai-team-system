# SOP-PM-01 项目启动

> 编号：SOP-PM-01 | 版本：V1.0 | 适用：合同签署 -> 章程批准

## 目的
规范从合同签署到章程批准的全部关键步骤。

## 前置条件
- [ ] 合同已签署并归档
- [ ] 销售已在Notion注册表录入，状态「已签约」
- [ ] PMO确认PM指派
- [ ] WF-01已触发

## 流程
合同签署 -> 销售填注册表 -> [WF-01自动]章程+Asana -> PM审阅 -> 内部启动会 -> 对外启动会 -> 客户批准章程 -> 进入执行

## 步骤
1. **合同信息归档(T+0)**: 销售填写Notion注册表，状态改为「已签约」
2. **PM接收与核查(T+1)**: 审阅Gemini生成的章程初稿，补充团队/里程碑/风险
3. **内部启动会(T+3)**: 45-60min，WBS核对/角色分工/技术预判
4. **对外启动会(T+7)**: 1-2h，项目目标/团队介绍/里程碑/沟通机制
5. **章程审批(T+14)**: 客户确认签字，注册表改为「交付中」

## 关键检查点
| 检查项 | 时间 | 负责人 |
|--------|------|--------|
| 注册表已填写完整 | T+0 | 销售 |
| Asana交付项目已创建 | T+0 | WF-01(自动) |
| 章程已审阅 | T+1 | PM |
| 内部启动会已召开 | T+3 | PM |
| 对外启动会已召开 | T+7 | PM |
| 客户已确认章程 | T+14 | PM+客户 |

## 异常处理
- WF-01未触发 -> 手动触发n8n Webhook
- 客户未签字 -> T+14升级给上级
- 资源不到位 -> 提交资源变更申请至PMO
